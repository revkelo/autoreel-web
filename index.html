How it works

From idea to published, unattended

Every video moves through a state machine — IDEA → GENERATING → READY → PUBLISHED — processed by a background worker, one video at a time.

STEP 01

Idea & script

A topic becomes a script with a viral hook in the first second, structured for retention. The scene design is a declarative project_json: layers with keyframes, real brand logos, cards and motion.

STEP 02

Voice & subtitles

Neural text-to-speech generates the voice-over locally, with word-level timing cues. Subtitles are laid onto the video automatically from those same cues — always in sync.

STEP 03

Render

A layer-based motion engine draws every frame on canvas — text, shapes, images, easing curves — and encodes MP4 with WebCodecs in headless Chromium. Vertical 1080×1920 or horizontal 1920×1080. Background music is mixed in automatically.

STEP 04

Thumbnail

Each video gets a designed thumbnail — adaptive title pill, real logos over clean cards — either from a custom design or generated automatically.

STEP 05

Publish

Finished videos are uploaded straight to the owner's own channels: YouTube (Data API v3 — vertical uploads become Shorts) and TikTok (Content Posting API, direct file upload). Publishing preferences and privacy level are configurable in-app.

STEP 06

Measure

Engagement stats for the owner's published videos — views, likes, comments — are snapshotted periodically and shown on a dashboard to decide what to make next.

Capabilities

What the studio can do

Engine

Layer-based motion engine

Text, shapes and images animated with per-property keyframes and easing curves. Auto-fit text inside cards, rounded corners, strokes, opacity choreography.

Audio

Local neural TTS

Voice-over synthesized locally with neural voices, tuned prosody, and precise timing cues that drive both subtitles and scene changes.

Formats

Vertical & horizontal

Shorts/TikTok-style vertical pieces (15–45s, one concept per video) and long-form horizontal formats like 5-minute quiz videos with segmented timers.

Templates

Reusable scene templates

Formulaic videos (top-5 lists, quizzes) instantiate from code templates in seconds — no design cost. Custom layouts are designed per-video when it's worth it.

Automation

Queue & worker

A scheduler claims queued videos and produces them end-to-end, unattended: TTS, subtitles, music, render, thumbnail, publish. Failures are retryable.

Agent-ready

MCP server

An MCP (Model Context Protocol) server exposes the whole studio as tools — create, validate, edit and render projects — so an AI agent can operate it conversationally.

Under the hood

Tech stack

Everything runs on the owner's own infrastructure. Rendering and voice cost nothing per video — no paid APIs in the production pipeline.

Web & API

  • Next.js (App Router) — dashboard + REST API
  • TypeScript — end to end
  • React — video library, planning, stats UI

Render engine

  • Canvas 2D — layer drawing contract
  • WebCodecs — MP4 encoding in-browser
  • Headless Chromium — unattended renders
  • edge-tts — local neural voice-over

Data & jobs

  • PostgreSQL — videos, projects, publications
  • Node.js worker — render & publish queue
  • Docker Compose — web + worker + db

Integrations

  • YouTube Data API v3 — resumable uploads, Shorts
  • TikTok Content Posting API — direct post
  • MCP server — AI-agent orchestration
  • Vitest + Playwright — tests & e2e
